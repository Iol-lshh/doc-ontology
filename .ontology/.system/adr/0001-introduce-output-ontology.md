# ADR 0001: 산출물 온톨로지 도입

- Status: Accepted
- Date: 2026-06-19

## Context

최종 산출물(코드·문서)에 근거·과정·이력이 섞여 verbose해진다. 근거를 지우면 추적성을 잃고, 남기면 인용 출처로서의 단정성을 잃는다. 두 요구가 충돌한다.

## Decision

산출물을 **본체(최종 산출물, 결론만)**와 **부속(근거·예제·이력)**으로 분리하고, 둘을 그래프로 연결하는 온톨로지를 둔다. 본체는 깨끗하게 유지하고, 근거는 부속 노드로 옮겨 추적성을 보존한다.

## Consequences

- 본체는 결론만 담아 인용 가능한 출처가 된다.
- 근거는 사라지지 않고 그래프로 추적된다.
- 관리 비용(노드 작성·빌드)이 새로 생긴다 → 스크립트로 자동화([[0008]]).
