# ADR 0002: 노드 식별자는 UUIDv7

- Status: Accepted
- Date: 2026-06-19

## Context

노드 간 관계의 앵커가 필요하다. 파일명(basename)을 앵커로 쓰면 파일을 옮기거나 이름을 바꿀 때 모든 참조가 끊긴다.

## Decision

노드에 `id`를 부여하고 관계(graphIndex)는 id로만 참조한다. 핵심은 이동·개명에 불변인 안정 앵커다.

**Concept·Instance(.md 노드)** — **UUIDv7 랜덤** 발급. 빌드가 빈 id를 채워 `database/` 원본 frontmatter에 되쓰고([[0006]]), 재빌드 시 그 값을 재사용한다. 저장된 id이므로 파일을 옮기거나 이름·내용을 바꿔도 **불변**이다 → 세대 간 추적·버전 이력([[0010]])·diff([[0012]])가 성립한다.

**Class(폴더)** — frontmatter가 없어 id를 저장할 곳이 없다. `relPath` 해시로 **결정적 생성**한다(UUID 형식 유지, 값은 경로로 정해짐). 같은 폴더는 항상 같은 id라 재빌드에 안정하지만, **폴더를 옮기면 id가 바뀐다**(이동 불변을 포기한 예외). 폴더 이동은 드물고 의미 변경([[0005]])이므로 감수한다.

해시 id로 .md 노드까지 통일하지 않는 이유: 경로/내용 해시는 이동·수정 시 id가 바뀌어 세대 간 추적이 깨진다. 그것이 UUIDv7+저장을 택한 이유다.

## Consequences

- .md 노드는 이동·개명·수정에도 id가 보존되어 관계·이력이 끊기지 않는다.
- Class는 저장 없이 안정한 대신 폴더 이동에 취약하다(트레이드오프).
- id는 사람이 읽기 어렵다 → 사람은 label로 작성하고 빌드가 변환한다([[0006]]).
- id 발급·중복 검사를 빌드가 자동화한다.
