# ADR 0003: 입력 영역과 시스템 영역 분리

- Status: Accepted
- Date: 2026-06-19

## Context

사람이 작성하는 소스와 빌드가 만드는 산출물이 한 곳에 섞이면, 사람이 산출물을 손대 깨뜨리거나 빌드가 소스를 덮어쓸 위험이 있다.

## Decision

두 영역으로 나눈다.

- **`database/`** — 사람이 편집하는 유일한 영역. 본문과 label 관계를 작성한다.
- **`.system/`** — 빌드 산출물 전용. 정규화 노드(`.system/database/`), 인덱스 3종, 스크립트. **사람은 건드리지 않는다.**

## Consequences

- 사람의 작업 표면이 `database/` 하나로 단정된다.
- `.system/`은 언제든 빌드로 재생성 가능한 파생물이다.
- 단, 빌드는 자동 채움 값을 `database/` 원본에 되쓴다(예외, [[0006]]).
