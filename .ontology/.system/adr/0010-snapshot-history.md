# ADR 0010: 빌드 스냅샷 이력으로 복원과 버전 이력을 통합

- Status: Accepted (저장 메커니즘은 [[0014]]가 대체 — git 객체 모델)
- Date: 2026-06-19

> 복원 정책(빌드 롤백/히스토리 롤백, backup, history.enabled)은 유효하다. 단 "세대별 디렉터리 통째 복사" 저장 방식은 [[0014]]의 git 객체 저장으로 대체됐다.

## Context

두 가지 이력 요구가 있다. (1) 빌드가 깨졌을 때 과거 정상 시점으로 **복원**, (2) 노드의 **버전 이력**(historyId/version, [[0004]]) 추적. `.ontology`는 git 저장소 안에 있으나, git에 위임하면 이 디렉터리만 따로 커밋·복원하기 번거롭고 레포 히스토리를 오염시킨다.

## Decision

**자체 스냅샷**으로 두 요구를 통합한다. 빌드 통과 시([[0008]]) 그 시점의 `.system/database` 전체를 `.system/database/history/<세대>/`에 적재한다.

- 각 스냅샷은 그 시점의 완전한 그래프다(선형, 빌드 통과 순서).
- 텍스트 md라 용량이 작아 **세대 제한 없이 전부 보존**한다(정리 로직 없음).
- 적재 여부는 `config.yml`의 `history.enabled`([[0009]])로 on/off.

복원은 두 기능으로 분리한다.

| 기능 | 대상 | 히스토리 off일 때 |
|---|---|---|
| **빌드 롤백** | 직전 통과 여벌(`.system/database/backup/`)로 복구. `database/` 원본은 보존. | 가능 |
| **히스토리 롤백** | 선택한 과거 세대 스냅샷(`history/<세대>`)으로 복원. | 불가 |

- **직전 여벌(backup)** — 빌드는 `.system/database`를 덮어쓰기 전에 그 직전 상태를 `backup/`에 1세대만 보관한다. 빌드 롤백은 이 backup을 복구한다. history와 무관하게 항상 1세대가 있으므로 history off여도 빌드 롤백이 가능하다.
- **세대 스냅샷(history)** — 무제한 누적([[0009]] on일 때). 히스토리 롤백은 그중 선택한 세대를 복구한다.

복원은 `.system`만 바꾸고 `database/` 원본은 건드리지 않는다. 노드 버전 이력은 그 노드가 들어 있던 과거 스냅샷에서 조회한다.

## Consequences

- 이력 메커니즘이 git과 무관하게 자기완결적이다.
- 선형 모델이라 분기·머지는 없다(빌드 통과 한 줄).
- 노드 단위 diff는 스냅샷 간 비교로 얻는다.
- 히스토리를 꺼도 빌드 롤백(여벌 1세대)은 항상 가능하다.
