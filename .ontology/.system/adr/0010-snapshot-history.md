# ADR 0010: 빌드 스냅샷 이력으로 복원과 버전 이력을 통합

- Status: Accepted
- Date: 2026-06-19

## Context

두 가지 이력 요구가 있다. (1) 빌드가 깨졌을 때 과거 정상 시점으로 **복원**, (2) 노드의 **버전 이력**(historyId/version, [[0004]]) 추적. `.ontology`는 git 저장소 안에 있으나, git에 위임하면 이 디렉터리만 따로 커밋·복원하기 번거롭고 레포 히스토리를 오염시킨다.

## Decision

**자체 스냅샷**으로 두 요구를 통합한다. 빌드 통과 시([[0008]]) 그 시점의 `.system/database` 전체를 `.system/database/history/<세대>/`에 적재한다.

- 각 스냅샷은 그 시점의 완전한 그래프다(선형, 빌드 통과 순서).
- 텍스트 md라 용량이 작아 **세대 제한 없이 전부 보존**한다(정리 로직 없음).
- 적재 여부는 `config.yml`의 `history.enabled`([[0009]])로 on/off.

복원은 두 기능으로 분리한다.

| 기능 | 대상 | 히스토리 off일 때 |
|---|---|---|
| **빌드 롤백** | 직전 통과 `.system` 여벌([[0008]])로 복구. `database/` 원본은 보존. | 가능 |
| **히스토리 롤백** | 선택한 과거 세대 스냅샷으로 복원. | 불가 |

노드 버전 이력은 그 노드가 들어 있던 과거 스냅샷에서 조회한다.

## Consequences

- 이력 메커니즘이 git과 무관하게 자기완결적이다.
- 선형 모델이라 분기·머지는 없다(빌드 통과 한 줄).
- 노드 단위 diff는 스냅샷 간 비교로 얻는다.
- 히스토리를 꺼도 빌드 롤백(여벌 1세대)은 항상 가능하다.
