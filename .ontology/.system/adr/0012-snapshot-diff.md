# ADR 0012: 스냅샷 비교 (그래프 + 파일)

- Status: Accepted
- Date: 2026-06-19

## Context

스냅샷 이력([[0010]])이 쌓이면 "두 시점이 무엇이 다른가"를 보고 싶다. 비교 대상은 과거 세대끼리, 그리고 과거 세대와 현재 상태(`.system/database`)다.

## Decision

스냅샷 비교를 두 층위로 제공한다.

- **그래프 diff** (먼저 보임) — graphIndex([[0007]]) 기준 노드·엣지의 추가/삭제/변경. "개념·관계가 어떻게 바뀌었나". 온톨로지 관점의 결론.
- **파일 diff** (드릴다운) — 해당 노드 `.md` 본문 단위 변경.

비교 대상은 세 가지다 — **세대**(commit sha, [[0014]]), **현재**(`.system/database`, `current`), **직전 여벌**(`backup`, [[0010]]). 임의의 두 대상을 비교한다.

| A | B | 의미 |
|---|---|---|
| 세대1 | 세대2 | 과거 두 시점 차이 |
| 세대N | 현재 | 그때 대비 지금 |
| 현재 | 직전 여벌(backup) | 마지막 빌드가 무엇을 바꿨나 |

현재·backup은 git 객체가 아니므로 비교 시점에 임시 tree로 떠서([[0014]]) 세대(commit의 tree)와 같은 방식으로 비교한다(객체는 쓰되 세대는 늘리지 않음).

비교 엔진은 하나이며 GUI/CLI가 대칭으로 호출한다([[0011]]). CLI는 텍스트 diff, GUI는 시각 diff.

## Consequences

- 그래프 수준 변화를 한눈에 보고 본문까지 파고들 수 있다.
- 히스토리가 off([[0009]])면 세대가 없어 세대 비교는 불가하다. 단 **현재↔backup 비교는 항상 가능**하다(backup은 history와 무관, [[0010]]) — "마지막 빌드의 변경"은 off여도 본다.
- 비교는 읽기 전용 — 엔진 상태를 바꾸지 않는다([[0008]]).
